# 第 3 阶段：安全闭环

- 登录密码和交易密码独立 BCrypt 存储，注册和修改时执行复杂度校验。
- 登录失败计数和短期锁定已实现；登录接口、转账预校验有内存限流。
- 修改登录密码后提升 `tokenVersion`，旧 token 自动失效；退出登录写入短期 token 黑名单。
- 身份证号、手机号加密存储，并保存 HMAC hash 用于唯一性查询。
- 完成敏感操作审计：登录、退出、查看/验证、修改密码、修改手机号、转账预校验/确认/执行、导出流水、账户操作等。
- 完成规则风控：单笔限额、凌晨敏感时段、大额陌生收款人、REJECT/CHALLENGE/MANUAL_REVIEW 动作。
- 大额/中风险交易预留短信/OTP，demo 验证码为 `000000`。

验证：`mvn -q test` 通过；测试验证旧 token 失效、越权拒绝、并发扣款余额不为负。
