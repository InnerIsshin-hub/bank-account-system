# 网上银行系统完善 TODO

> 项目现状：当前仓库是 Spring Boot 3 + MyBatis-Plus + MySQL 后端、Vue 3 + Vite + Element Plus 前端的网银 demo。已具备注册、登录、账户列表展示雏形、密码校验弹窗等基础页面，但大量数据仍为模拟数据，核心转账、余额查询、交易流水、统一异常、鉴权、安全审计、高并发组件、智能 Agent 与风控能力尚未闭环。

## 0. 任务优先级约定

- **P0 必须完成**：让 demo 具备可运行、可演示、数据闭环、资金不出错的基础能力。
- **P1 核心完善**：补齐一个较完整网银系统应有的用户、账户、交易、安全、通知和后台管理能力。
- **P2 架构增强**：引入 MQ、Redis、分布式事务、分库分表、监控审计等高并发金融系统组件。
- **P3 智能化能力**：落地银行 Agent、智能账单分析、反欺诈模型、KYC 生物识别等能力。

## 11. 当前代码中最应优先修复的点

- [ ] `AccountServiceImpl` 中 `deposit`、`withdraw`、`transfer`、`getBalance` 仍是 `UnsupportedOperationException`，必须作为第一批实现。
- [ ] `RecordController` 为空，交易流水无法查询。
- [ ] `BusinessException` 为空，异常体系没有建立。
- [ ] `MyBatisPlusConfig` 为空，分页、自动填充、乐观锁尚未配置。
- [ ] 前端 `Dashboard.vue`、`Transfer.vue`、`Records.vue` 仍有模拟账户和模拟流水，需要接真实接口。
- [ ] 当前登录只存 `localStorage.userInfo`，没有 token 和后端鉴权，存在越权风险。
- [ ] 当前转账 DTO 中 `tradePassword` 标注“可先忽略”，与银行系统安全要求冲突。
- [ ] 卡号查看已有密码校验弹窗雏形，但账户数据是模拟的，且完整卡号会进入前端内存，需要后端按需返回并审计。
- [ ] Controller 层大量 `try/catch` 返回 `Result.error(e.getMessage())`，容易泄露内部错误信息。
- [ ] 缺少测试，尤其是并发扣款、余额不足、幂等重复提交、越权访问测试。
